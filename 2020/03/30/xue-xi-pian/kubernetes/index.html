Iaas

  • infrastructure as a service
  • 基础设施即服务
  • 对应计算机的物理机
  • 比较流行的产品有openstack、VM等

Paas

  • platform as a servie
  • 平台即服务
  • 现今业界公认的标准产品是Docker
  • 针对于Iaas进行上一层的封装

Docker

  • 将环境容器化

    • 带来的问题是容器与外界访问被容器本身封装了一层，需要增加容器端口和物理机端口的资源映射完成和外界通信，效率变低

  • 安装

    1. SET UP THE REPOSITORY
      • yum install -y yum-utils device-mapper-persistent-data lvm2
    2. 替换yum的docker源
      • wget https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo -O /etc/yum.repos.d/docker-ce.repo
    3. INSTALL DOCKER ENGINE - COMMUNITY
      • yum install docker-ce docker-ce-cli containerd.io
    4. 镜像加速器
      • https://cr.console.aliyun.com/
        sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
"registry-mirrors": ["https://7som5skl.mirror.aliyuncs.com"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

  • 搭建本地私服

    • docker run -d -p 5000:5000 –restart always –name registry registry:2

      1. docker pull registry
      2. docker run -d -p 5000:5000 –restart always –name registry
      3. docker tag ubuntu localhost:5000/ubuntu:v1
      4. docker push localhost:5000/ubuntu:v1
      5. docker pull localhost:5000/ubuntu:v1

    • 设置私服访问权限

        ## 创建目录
  mkdir -p /data/DockerRegistry/{Data,Cert,Auth}
  ## 生成TLS使用的自签名证书
  cd /data/DockerRegistry/Cert

  domain=MyDockerRegistry.com
  openssl req -nodes \
  -subj "/C=CN/ST=Hubei/L=Wuhan/CN=$domain" \
  -newkey rsa:4096 -keyout $domain.key -out $domain.csr
  openssl x509 -req -days 3650 -in $domain.csr -signkey $domain.key -out $domain.crt

  ## 生成鉴权使用的密码文件
  docker run --name once --entrypoint=htpasswd registry:2 -Bbn docker docker666 > /data/DockerRegistry/Auth/Htpasswd
  docker rm once

  ## 运行容器
  docker run -d \
  -p 5000:5000 \
  --restart=always \
  --name registry \
  -v /data/DockerRegistry/Data:/var/lib/registry \
  -v /data/DockerRegistry/Cert:/Cert \
  -v /data/DockerRegistry/Auth:/Auth \
  -e "REGISTRY_AUTH=htpasswd" \
  -e "REGISTRY_AUTH_HTPASSWD_REALM=Registry Realm" \
  -e REGISTRY_AUTH_HTPASSWD_PATH=/Auth/Htpasswd \
  -v "$(pwd)"/certs:/certs \
  -e REGISTRY_HTTP_TLS_CERTIFICATE=/Cert/MyDockerRegistry.com.crt \
  -e REGISTRY_HTTP_TLS_KEY=/Cert/MyDockerRegistry.com.key \
  registry:2

  ## 配置域名
  echo "127.0.0.1 MyDockerRegistry.com" >> /etc/hosts
  ## 查看docker registry中的镜像
  curl https://MyDockerRegistry.com:5000/v2/_catalog -u docker:docker666 --cacert /data/DockerRegistry/Cert/MyDockerRegistry.com.crt

k8s

  • 分布式资源管理框架
  • 优点
    • 开源
    • 弹性伸缩
    • 轻量级
    • 负载均衡：ipvs策略
  • 前身
    • google borg框架的go语言版本
  • 组件
    • master
      • kubectl：命令终端
      • web UI：页面控制台
      • scheduler：调度中心，选择合适的node完成任务分配
      • controller：控制中心，维护副本期望数目
        • replication controller
        • replicaSet controller
          • 支持HPA（horizontal pod autoscale），通过配置条件自动扩缩容
        • deployment controller
          • 支持HPA
        • 高可用集群副本数目最好是大于1的奇数，原因同zookeeper选举机制，大于半数策略
        • 新版本中增加了replicaSet，该控制器支持组合式操作pod，比如通过pod的别名+pod版本来操作pod，RC不支持组合式
        • 滚动更新则需要使用deployment控制器，原理是它会生成RS来创建pod，如果需要更新pod版本，则会创建新的RS_NEW来创建新的pod，同时废弃旧版本的pod
        • statefullSet
          • 解决有状态服务问题
          • 稳定的持久化存储
            • pod重新调度后能访问相同的持久化数据，基于PVC实现
          • 稳定的网络标志
            • pod重新调度后能保持原有的podname和hostname等，基于Headless Service
          • 有序部署
          • 有序收缩
        • daemonSet
          • 确保全部或部分node上运行一个pod副本
          • 当有node加入则自动创建pod，node移除会自动销毁pod
          • daemonset被删除时，会自动删除它所创建的所有pod
        • job，cronjob
      • api server：服务中心，所有服务访问的统一入口
      • etcd：k-v分布式存储服务
    • node
      • kubelet：与cri（container runtime interface）交互，维持pod的生命周期
      • kube proxy：写入规则到iptables、ipvs，负责pod的负载均衡，主要是与防火墙交互进行映射处理
        • firewall
      • pod
        • container：主流的docker
        • 自主式pod
        • 被控制管理的pod
        • 同一个pod中的各个容器交互
          • io
          • 公用pause网络栈，通过localhost就可以相互访问
        • pod和pod的交互
          • overlay network
            • flannel开源产品实现
          • 1:1或N:1
            • k8s自身就支持
          • 1:N或N:N
            • 可以通过SVC（service）桥接来实现
        • pod与svc交互
          • iptables或ipvs
  • 插件
    • coreDNS：可以为集群中的svc创建域名ip关系解析
    • dashboard：为k8s提供一个B/S的访问模式
    • ingress controller：官方只能提供4层代理，它可以提供7层
    • fedetation：提供一个可以跨集群中心多k8s统一管理功能
    • prometheus：提供k8s监控能力
    • ELK：提供k8s日志统一分析接入平台
